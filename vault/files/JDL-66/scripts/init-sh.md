Section A â€” pre-flight dependency checks â€” is idempotent and non-mutating. `check_docker_compose_v2()` verifies `docker` is on `PATH`, then that `docker compose version` responds, then that the reported version is v2.x or greater; the major-version accept pattern is a glob on `docker compose version --short` that accepts `2.x`, `3.x+`, and `10+.x` so the check survives future Compose major-version bumps without code changes. `check_repo_root()` verifies the sentinel trio `compose.yml` + `.env.example` + `Makefile` is all present in the current working directory, and names the specific missing file(s) in any failure message so an operator who ran the script from the wrong directory sees exactly which sentinel(s) were absent rather than a generic "not in repo root" error. Section B â€” ephemeral artifact generation â€” mutates state and runs only if Section A passed. `generate_env()` checks for `.env`. The tooling surface is POSIX-shell-only: `bash`, `tr`, `sed`, `grep`, `head`, `printf`, `command`, `cp`, `[`, `case`, and `/dev/urandom`. Forge modified `scripts/init.sh` at commit `075cb6b` on branch `JDL-66-m1-shared-infrastructure-online`, scoped exactly to three lines inside the `generate_env()` function's `while grep -q '=CHANGEME$' .env; do` loop â€” the random-secret generation pattern that was blocking `make up`. The change fixes the SIGPIPE-under-`pipefail` defect diagnosed in [[incidents/JDL-66/20260424T1618Z-sigpipe-under-pipefail-in-scripts-init-sh]] and reproduced 3/3 by the M1 smoke Probe run (task `a468e17a6a5359be5`, tests at [[files/JDL-66/app/tests/test_m1_infrastructure_smoke-py]], commit `6fbad22`). ^p006

Before: two lines â€” `local rand` + `rand=$(tr -dc 'A-Za-z0-9' </dev/urandom | head -c 32)`. This pipeline fails under `set -euo pipefail` because `head -c 32` closes stdin after 32 bytes, `tr` receives SIGPIPE on its next write attempt while reading from the infinite `/dev/urandom`, exits 141, `pipefail` propagates the 141 as the pipeline status, and `set -e` terminates the script before `sed` ever runs â€” so `.env` lands with every `CHANGEME` slot unsubstituted, matching the failure signature recorded in the sibling incident. ^p007

After: three lines â€” `local rand chunk` + `chunk=$(LC_ALL=C tr -dc 'A-Za-z0-9' < <(head -c 256 /dev/urandom))` + `rand="${chunk:0:32}"`. The bounded-read pattern: `head -c 256 /dev/urandom` reads exactly 256 bytes and exits cleanly (it is reading, not writing to a closed pipe), `tr` consumes those bounded bytes via process substitution, filters to ~170 alphanumerics on average (~66% keep rate from 62-of-94 printable retention), and exits normally â€” no pipeline-to-premature-closer anywhere, so no SIGPIPE. Bash then slices the first 32 characters via `${chunk:0:32}`. `LC_ALL=C` keeps `tr` in single-byte mode regardless of operator locale. Entropy margin is comfortable â€” 170 alphanumeric chars available, 32 consumed. ^p008

Per-iteration independence recorded in the initial entry's ^p003 is preserved â€” each pass through the `while grep -q '=CHANGEME$' .env; do` loop generates a fresh `chunk` and derives a fresh `rand`, so every `CHANGEME` slot still receives an independent random 32-char value per the `.env`-auto-generation contract recorded in [[decisions/JDL-66/20260424T1519Z-up-sh-auto-generates-env-on-first-boot]]. Forge smoke-tested the new pattern three times in isolation under `set -euo pipefail`; each produced a 32-character string with no exit-141. ^p009

Everything else in the file recorded in the initial entry is preserved verbatim: `set -euo pipefail`, `check_docker_compose_v2()` and its major-version accept-glob (initial ^p002), `check_repo_root()` and its sentinel-trio check (initial ^p002), the `log()` / `fail()` helpers, the `main()` ordering that runs Section A before Section B (initial ^p001), the header comment block, and the `sed -i "0,/=CHANGEME$/ s//=${rand}/" .env` substitution line itself â€” only the two-line random generator becomes three lines, and the identifier `rand` consumed by `sed` stays identical.
