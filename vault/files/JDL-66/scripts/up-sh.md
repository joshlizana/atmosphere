## Purpose
Tracks the lifecycle of `/home/josh/atmosphere/scripts/up.sh` — the tiered, health-gated operator bring-up script for the Atmosphere platform. Brings the stack up from a clean state in dependency order, polling each tier's docker healthchecks to gate progression before the next tier is started. Designed as the canonical one-command entry point that future milestones extend by appending tier blocks rather than by restructuring.

The file is committed executable (mode `100755`) and tracked in git so `git clone && ./scripts/up.sh` works on a fresh checkout without a separate `chmod +x` step. ^p001

The preflight block verifies Docker Compose v2 is available on PATH, creates `./data/seaweedfs` and `./data/redpanda` if absent, and `chown`s them to their container UIDs (`1000:1000` for SeaweedFS and `101:101` for Redpanda) via `sudo`. The chown is load-bearing because a fresh bind-mount directory is owned by root on creation and the SeaweedFS (UID 1000) and Redpanda (UID 101) processes fail to write to their data dirs without it — the failure mode is a confusing permission-denied at container start rather than an obvious missing-directory error. Postgres is explicitly excluded from the chown step because it uses a docker-managed named volume (`atmosphere_postgres`) whose permissions are handled by docker itself, keeping Postgres's strict `0700`/`0750` data-dir check cleanly satisfied without operator intervention. ^p002

A `wait_for_healthy <timeout-seconds> <svc>...` shell helper polls `docker inspect --format='{{.State.Health.Status}}' ${PROJECT}-<svc>-1` on a 2-second cadence until every named service reports `healthy` or the timeout fires. On timeout it dumps `docker compose logs --tail=50` for each still-pending service before exiting non-zero, so the operator gets actionable diagnostic output instead of a bare timeout message. Services without a healthcheck are treated as a config bug and fail loudly rather than being silently allowed to pass — this keeps healthcheck coverage a tier-gating invariant rather than a best-effort signal. ^p003

Tier 1 (M1) runs `docker compose up -d postgres seaweedfs redpanda`, waits for all three to report `healthy`, then runs `docker compose up --exit-code-from seaweedfs-init seaweedfs-init` in the foreground as a one-shot so the bucket-creator's exit status surfaces to the script and any bucket-provisioning failure aborts the bring-up rather than being silently swallowed in a `-d` background run. The main function contains explicit comments marking where future tiers land — M2 Lakekeeper, M3 observability fabric, M4+ data-plane services — so each milestone extends the script by appending to a reserved slot rather than by restructuring the control flow. ^p004

The `COMPOSE_PROJECT_NAME` prefix is honored via env with a default of `atmosphere`, so the script works under custom project names without code changes; this matches the `COMPOSE_PROJECT_NAME=atmosphere` pin documented in `.env.example` (see `[[env-example]]`) and keeps the container-name construction (`${PROJECT}-<svc>-1`) stable across operator checkout-directory variations. ^p005

Forge modified `scripts/up.sh` at commit `1656700` on branch `JDL-66-m1-shared-infrastructure-online` with a net-smaller diff (+26/-31) that lands two coupled reworks matching the M1 volume-strategy shift. The changes complete an architectural pivot decided upstream in `[[decisions/JDL-66/20260424T1518Z-docker-named-volumes-for-state]]` (all state on docker-managed named volumes, no bind mounts) and `[[decisions/JDL-66/20260424T1519Z-up-sh-auto-generates-env-on-first-boot]]` (the script materializes `.env` on first boot). Both changes land together because one obsoletes the preflight that the other replaces. ^p006

First, a first-boot `.env` auto-populate block is inserted immediately after `set -euo pipefail`. If `.env` is absent, the script copies `.env.example` to `.env` and then replaces every `=CHANGEME` placeholder with an independent URL-safe 32-character random string. The implementation uses a loop-until-no-`CHANGEME` pattern: `tr -dc 'A-Za-z0-9' </dev/urandom | head -c 32` generates a fresh random for each iteration, and `sed -i "0,/=CHANGEME$/ s//=${rand}/"` replaces only the first remaining occurrence per pass so each slot receives its own independent random value. Uniform substitution — a single random reused across every slot — is explicitly avoided. The block is idempotent: if `.env` already exists it is silently skipped with no regeneration and no output. One log line fires when generation runs; zero secret values are ever printed to stdout or stderr. Operator-initiated regeneration requires `rm .env` first, so existing operator customization always wins. ^p007

Second, the bind-mount preflight is removed entirely. The `mkdir -p data/seaweedfs data/redpanda` lines and their associated `sudo chown 1000:1000` / `sudo chown 101:101` blocks are gone, along with the UID-1000 (SeaweedFS) and UID-101 (Redpanda) explanatory comments that justified them in the initial landing recorded in the `2026-04-24T12:37Z — initial` entry above. Those lines existed because fresh bind-mount directories are owned by root on creation and the container processes fail to write to their data dirs without a pre-`chown` — that failure mode surfaces as a confusing permission-denied at container start rather than an obvious missing-directory error. Under the named-volumes strategy encoded in `[[files/JDL-66/compose-yml]]`, every stateful service persists to a docker-managed named volume whose ownership docker handles automatically on first attach, so the pre-`chown` is no longer needed and the `sudo` dependency disappears from the script's assumption set. `preflight()` narrows to a single responsibility: verifying `docker compose` v2 is available on PATH. ^p008

The header comment block is updated to match the new surface. The passwordless-sudo / chown rationale is dropped, a short paragraph describing the `.env` auto-generation semantics is added (the `CHANGEME`-replacement contract plus the "operator customization wins" guarantee), the assumptions block narrows to "Docker Compose v2 on PATH" + "cwd is the repo root", and the exit-code-1 description is revised to reflect the removed failure modes. Everything else the initial landing recorded is preserved verbatim: `wait_for_healthy`, the `log` / `warn` / `fail` helpers, the `PROJECT` and `DEFAULT_TIMEOUT` configuration, the `tier1_m1_infra` block that brings up Postgres / SeaweedFS / Redpanda and runs the `seaweedfs-init` one-shot, the future-tier marker comments in `main()` that reserve slots for M2 Lakekeeper / M3 observability / M4+ data-plane services, and the `COMPOSE_PROJECT_NAME` defaulting logic documented in the initial entry's `^p005`. ^p009

`bash -n scripts/up.sh` passes without error and the file mode remains `100755`, so the commit preserves both syntactic validity and the commit-executable invariant the initial landing established. The net result is a smaller, simpler script whose preflight responsibility has shrunk to just the Compose-v2 availability check and whose first-boot contract now produces a ready-to-use `.env` without a separate `cp .env.example .env && $EDITOR .env` step for the operator. ^p010
